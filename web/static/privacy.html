Privacy by architecture

作品默认留在你掌控的环境。

以下说明区分静态 Pages、浏览器本地数据、本地后端和第三方模型服务，避免把“本地优先”误解成“任何操作都不联网”。

GitHub Pages

Pages 静态页面本身不提供写作工坊账号系统或常驻后端。页面可以在浏览器 IndexedDB 保存本地项目；GitHub 作为托管平台仍可能按其政策记录常规访问日志。

浏览器与本地后端

浏览器数据

项目、章节与界面状态可保存在当前站点的浏览器存储中。清除站点数据会删除它们，请定期导出。

服务端数据

完整本地模式会把配置与项目数据写入你指定的工作目录。默认仅监听 127.0.0.1。

模型与能力请求

当你主动执行 AI 或 Skill 任务时，明确选择的上下文会先发送到你运行的后端，再由后端调用已配置的模型服务。第三方提供商如何保存与处理数据，取决于你与该提供商之间的条款。

密钥规则：API Key 应存放在本地配置或环境变量中。不要写进 GitHub Pages、前端 JavaScript、公开仓库、Issue 或截图。

第三方来源

登记 GitHub 仓库、Skill manifest 或远程链接不会自动执行其中代码。若未来启用代码型能力，应先核对来源、许可证、版本与权限，并使用隔离执行环境。

你可以做什么

  • 随时从浏览器或本地工作目录导出项目。
  • 删除浏览器站点数据或后端工作目录中的项目文件。
  • 使用环境变量保存密钥，并在服务商后台撤销密钥。
  • 运行前检查上下文包，不发送与本次任务无关的作品内容。

安全问题请按 SECURITY.md 报告；普通问题请使用 GitHub Issues。社区联系为 Linux DO · The_o0l。
