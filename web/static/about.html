Open-source writing infrastructure

让 AI 参与创作，但不接管作品。

Writing Workshop 是面向小说与长篇内容的本地优先工作台：资料、上下文、能力和写入动作都由作者明确控制。

我们在解决什么

长篇项目的难点不是“再生成一段”，而是维护角色、设定、章节、记忆与修改边界，同时避免上下文成本随资料库无限增长。

产品原则

显式上下文、候选先行、作者确认、写前快照、本地默认、能力可审计。模型能力可以替换，作者控制不能消失。

能力生态

工作台可以登记后端项目、Skill、规则包和提示能力，并把多个 Skill 保存为技能包。每项能力声明来源、版本、分类、步骤、权限与输出，不把链接误当可执行代码。

开放协作

项目采用 Apache-2.0 许可证，包含 Go 同源后端、原生 Web 前端、协议文档、CI、Docker、Pages 与发布工作流。

项目边界

浏览器使用同源 /api/ 与本地或自部署后端通信。保存 GitHub 链接不会执行第三方仓库；真实第三方代码执行需要独立的最小权限沙箱。静态 Pages 是可交互展示，不承诺常驻 API。
