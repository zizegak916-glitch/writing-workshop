主应用管理后台关于

使用文档

AI写作工坊是主体应用：GitHub Pages 发布静态前端，后端、GitHub 开源项目、skill 和自定义能力都只是可接入的能力来源。主应用默认以游客本地模式打开，不要求设置密码。

打开写作工坊 打开能力后台
当前 Pages 地址没有常驻后端。你可以浏览界面、使用本地浏览器数据和打开能力后台；真实 AI 调用、配置保存、项目同步、skill 执行和规则管理，需要运行本地后端或部署自己的 API 服务。

主体与能力来源

写作工坊统一保存和调度多个能力来源：GitHub 上的优质开源项目、可组合 skill、项目规则包，以及你自己编写的通用能力。项目拥有独立的 Go 模块、命令入口和同源后端；底层写作引擎的上游来源与许可证保留在仓库文档中。

执行链路

选择能力

前端允许选择一个后端项目，或同时选择多个 skill，并带上当前项目、章节、角色、上下文和用户参数。

后端执行

后端接收结构化请求，调用对应项目或 skill，输出文本、JSON、补丁、文件列表或事件流。

实时回传

后端输出直接传回前端。长任务应支持流式输出，前端可以随时中断并保留已生成内容。

主界面的“流程”怎么用

1. 写清本次任务

打开右侧 AI 面板的“流程”页，只描述这一次要完成的工作。点“新任务”会清空当前候选，但不会删除历史。

2. 选择上下文

按需勾选当前选区/正文、项目设定、大纲、人物卡和记忆。界面会显示字符数与估算 token；预览区只做界面截取，发送内容不会静默截断。

3. 组合能力

选择一个后端和需要的 skill。展开能力可以看到步骤、入口和权限；能力说明与执行要求可在后台编辑。

4. 生成候选

点击“生成候选”后通过同源 /api/run 流式返回。它不会自动覆盖编辑器；运行中可以随时中断。

5. 明确决定去向

候选可以复制、替换运行时选区、插入当前光标或追加到正文。正文在运行期间发生变化时，系统会阻止按旧坐标替换。

6. 记忆单独确认

“整理为记忆…”只会把候选送入记忆编辑框，仍需你再次确认保存。正文写入和记忆同步不会绑在一起。

流程历史保留完整候选；发生正文写入时还会记录写入前快照。你可以重新打开候选，或恢复到写入前。恢复动作本身不会自动保存章节。

当前可用接口

保存能力

GET/POST /api/capabilities 已可用，能力保存到后端工作目录的 .ainovel/capabilities.json。

执行能力

POST /api/run 已可用，支持 backend_id、skill_ids、任务、上下文和参数。

中断运行

POST /api/abort 会同时中断写作运行和能力运行；流式输出使用 SSE 的 start/delta/done 事件。

默认内置 builtin-echo、builtin-outline、builtin-rewrite 和 writing-workshop。内置能力只读；用户保存的能力可以在后台编辑说明、执行要求、可见步骤和权限边界，也可以启停、删除、流式试跑或中断。选中的 prompt/skill 执行要求会进入本次模型请求；保存 GitHub 链接或 skill manifest 本身不会执行任意仓库代码，第三方代码执行仍需要明确的沙箱执行器。

自定义文件或链接要求

添加 GitHub 项目、远程链接或本地 skill 时，必须能被后端安全解析，并满足下面的最小契约。

来源

提供 GitHub 仓库链接、版本或 commit；本地文件需包含名称、版本、作者、许可证和用途说明。

入口

声明可执行入口或提示词入口，例如命令、HTTP endpoint、脚本路径、prompt 模板或规则文件路径。

输入输出

明确输入 schema、输出类型、是否支持流式输出、是否支持取消，以及错误返回格式。

权限边界

声明需要访问的文件、网络、环境变量和密钥。默认不能越权读取用户项目之外的数据。

通用性

能力必须面向写作、分析、导入、改写、规划等通用任务，不能绑定某个单一小说或私有项目。

可审计

保存原始链接、版本、摘要和启用状态，方便用户知道当前前端传给后端的能力组合。

Writing Workshop 本地后端

go build -o writing-workshop ./cmd/writing-workshop
./writing-workshop serve --demo --port 8080

使用该后端示例时，本地服务启动后访问 http://127.0.0.1:8080/app.html 或 http://127.0.0.1:8080/admin.html。此时前端会通过同源 /api/ 端点连接后端。主应用是本地游客模式；除非后续提供数据管理服务，否则不要求用户设置账号密码。

核心入口

写作工坊

app.html，用于项目、章节、角色、AI 辅助写作。

管理后台

admin.html，用于配置 provider、model、API key、后端项目、skill、项目规则，并测试 /api/run。

API

后端端点以 /api/ 开头，主要包括 /api/ai、/api/config、/api/rules、/api/capabilities、/api/run。

开源自定义

模型供应商

在后台或配置文件中修改 provider、model、base URL，也可以通过环境变量注入 API key。

能力组合

保存多个后端项目和多个 skill，创作时按任务选择组合，由前端传递给后端执行。

前端页面

静态文件位于 web/static/，fork 后可以替换文案、样式、入口页面和能力管理界面。

API Key 配置

推荐在管理后台保存 provider 配置，或使用环境变量，例如 AINOVEL_OPENROUTER_API_KEY、OPENROUTER_API_KEY、OPENAI_API_KEY。
