Product guide · 2026-07-22

从打开页面，到完成一次安全写作。

这份文档不假设你已经懂 API、Docker 或 Skill。先选择你的使用方式，再按一条完整路线跑通。

先理解：Pages 在线版与后端增强模式

GitHub Pages · 正式在线版

这是项目当前对外发布的真实 HTTPS 站点，不是临时预览。可以正式建立浏览器本地项目、编辑正文、管理分类与导入导出。数据默认保存在当前域名的 IndexedDB / localStorage。

后端增强模式

运行仓库自带的 Go 服务，或部署兼容后端后，可增加服务端项目同步、密钥保管、Skill 执行、SSE 流式输出和中断。这是能力扩展，不是把 Pages 从“假站”变成“真站”。

托管方式不等于功能真假：静态网页同样可以发起 API 请求；是否成功取决于实际端点、认证和 CORS。本仓库当前默认 Pages 工作流未同时部署 Go API，所以界面显示“未连接后端”仅说明服务端能力未接入。GitHub Pages 与 OpenAI Sites 是独立托管方式。

60 秒启动

路线 0：直接使用 Pages 在线版

打开 Writing Workshop 工作台，新建项目即可编辑。无需安装 Go 或 Docker。由于项目数据属于当前 github.io 域名的浏览器存储，请定期使用顶部导出或项目操作台下载备份。

路线 A：Docker（最省事）

需要安装 Git 与 Docker Desktop。复制下面四行命令：

git clone https://github.com/zizegak916-glitch/writing-workshop.git
cd writing-workshop
docker compose up --build

# 浏览器打开
http://127.0.0.1:8080/app.html

第一次会以无密钥 Demo 模式启动。看到工作台后，打开 能力后台配置模型；也可以先不配，验证本地项目和内置能力链路。

路线 B：直接运行源码

需要 Go 1.25 或更高版本：

git clone https://github.com/zizegak916-glitch/writing-workshop.git
cd writing-workshop
go build -o writing-workshop ./cmd/writing-workshop
./writing-workshop serve --demo --port 8080

默认只监听 127.0.0.1，这是为了避免把作品与密钥意外暴露到局域网。如确实需要容器或局域网访问，再显式加 --host 0.0.0.0，并自行配置反向代理鉴权。

确认服务健康

curl http://127.0.0.1:8080/api/health
# 预期：{"mode":"demo","status":"ok"}

建立第一个项目

点击顶部 ＋填写项目名称添加第一章建立人物卡开始写作
  1. 在工作台顶部点击 ＋，填写项目名、题材与简介；已有自定义分类时可同时多选分类。
  2. 左侧切到“章节”，点击“添加章节”，输入标题。
  3. 切到“人物”，录入姓名、定位、描述与角色弧线。
  4. 在中间编辑区写作。自动保存开启时，数据写入浏览器 IndexedDB；点击顶部保存可以立即保存。
  5. 第一次重要编辑后，点击顶部下载图标导出项目备份。项目包版本 2 包含项目、章节、大纲、人物与记忆。
本地优先：Pages 正式在线版与后端增强模式都可以使用浏览器数据库。清除浏览器站点数据会删除这部分内容，因此长期项目务必定期导出。

工作台各区域做什么

区域用途常用操作
顶部栏项目切换、统计、全局设置新建、后台、设置、导出、保存
左侧资料栏项目结构与设定大纲、章节、人物、笔记
中央编辑区唯一正文工作面格式、专注、预览、快速 AI、导出
右侧 AI 区任务与候选助手、对比、记忆、历史、流程
能力后台服务端配置与调试Provider、URL、Key、项目、规则、Skill、API 测试

手机端会把资料栏和 AI 区折叠到底部导航。正文仍是默认首屏，不会把桌面三栏硬挤进窄屏。

项目操作台

点击顶部项目名进入“我的项目”。搜索框会匹配项目名、类型和自定义分类；分类下拉可进一步筛选。每个项目提供重命名、复制、分类、单独导出和删除。删除会明确列出连带删除的大纲、人物、章节与记忆并再次确认，不提供无确认批量删除。

“分类管理”创建浏览器本地分类并指定范围；项目和记忆分类都可自定义。它们保存在当前站点的浏览器存储，不会自动覆盖后台的能力分类。

完成一次 AI 写作流程

描述任务选择上下文组合能力生成候选确认写入
  1. 描述这一次任务。例如“检查这段对话的动机断点”，不要写成永久项目规则。
  2. 只选需要的上下文。当前选区/正文、项目设定、大纲、人物卡、记忆都可以单独开关；界面显示字符数和估算 token。
  3. 选择后端和 Skill。可逐项勾选多个 Skill，也可一键应用技能包；“已选 N 项”会明确显示当前组合。分类下拉只改变显示，不丢失其他分类中已选的 Skill。
  4. 点击生成候选。本地后端通过 POST /api/run 返回 SSE 事件；运行中可以中断。
  5. 检查后再决定去向。候选可复制、替换运行时选区、插入光标或追加到正文。正文若在生成期间改变，旧坐标替换会被阻止。
  6. 记忆另行确认。“整理为记忆”只把候选放入记忆编辑框，仍需再次保存；不会和正文写入捆绑。
可恢复：流程历史保留完整候选；发生正文写入时还会记录写入前快照。恢复不会自动保存章节，你仍有最后一次确认机会。

导入、导出与备份

导入单篇内容

在新建/导入入口选择 Markdown、TXT 或 DOCX。先检查解析后的标题与正文，再放入当前项目。

导出当前正文

编辑工具栏的“导出”用于保存正在编辑的章节，适合送审、分享或进入其他排版工具。

导出完整项目

顶部下载按钮或项目操作台“导出”会打包项目、章节、大纲、人物与记忆。长期创作建议每个重要阶段保存一份带日期的备份。

增量资料

新增资料应按文件 hash 只处理一次；脚本负责去重和分类，AI 只总结新文件，避免每次重读全库。

如果浏览器提示下载被阻止，请允许当前站点下载文件。若导入失败，先确认文件未加密，并用 UTF-8 保存纯文本或 Markdown。

配置 API 与模型

从 http://127.0.0.1:8080/admin.html 打开能力后台，在“配置”页填写：

字段示例说明
Provideropenai当前提供商名称，可自行命名
Typeopenai兼容类型：openai / anthropic / gemini
Modelgpt-4.1-mini必须是你的服务实际支持的模型 ID
Base URLhttps://api.openai.com/v1API 根地址，不要填 ChatGPT 网页地址
API Key••••••留空可保持原值或改用环境变量

保存后切到“API 测试”，先执行“模型直连测试”；成功后再用“执行能力”验证 Skill 组合、流式回传和中断。

使用环境变量（更适合长期部署）

export OPENAI_API_KEY="your-key"
./writing-workshop serve --port 8080

OpenRouter 也可使用 OPENROUTER_API_KEY。配置读取接口不会把已保存的完整密钥返回给浏览器。

为什么这里不让浏览器直连模型 API

CORS 不是“再加一个开关就能安全解决”的小错误。把 API Key 放在浏览器里，会被页面脚本、扩展或网络日志读到；不少模型服务也会拒绝来自任意网页的跨域请求。

浏览器工作台同源 /api本地 Go 后端模型服务候选返回

Writing Workshop 采用同源桥接：浏览器只访问自己的 /api/；后端读取密钥并请求模型，再把结果流式传回。这样既绕开浏览器跨域限制，也不向前端暴露密钥。

不要：在 GitHub Pages 前端文件、仓库 JavaScript、公开环境变量或可下载配置中写入真实 API Key。Pages 是正式在线站；正因为它真实公开，才更不应将密钥打包进前端。

添加 GitHub 项目或 Skill

在能力后台的“能力来源”页添加一项能力。至少填写名称、类型、版本、来源、许可证、入口、执行说明、可见步骤、权限边界和输出类型；分类与标签用于检索，不会增加权限。

{
  "name": "场景节奏检查",
  "type": "skill",
  "category": "revision",
  "tags": ["场景", "节奏"],
  "version": "1.0.0",
  "source": "https://github.com/example/scene-pacing",
  "license": "Apache-2.0",
  "entry": "prompt:scene-pacing",
  "output": "text",
  "steps": ["读取显式上下文", "检查节奏", "返回候选"],
  "permissions": ["context:read"],
  "supports_stream": true,
  "supports_abort": true
}

当前实现会登记、校验和组合能力清单，但不会因为你粘贴 GitHub URL 就执行仓库代码。第三方代码需要明确的沙箱执行器与最小权限机制；这是安全边界，不是缺少一个按钮。

创建技能包

  1. 先在“能力来源”确认需要组合的 Skill 都已启用。
  2. 切到“技能包”，填写名称、说明和分类。
  3. 每行或用逗号填写一个 Skill ID；保存时服务会去重并拒绝未知或停用 ID。
  4. 回到工作台“流程”页，点击技能包即可选中整组 Skill；你仍能逐项取消或补选。

创建后台分类

“分类”页支持名称、颜色、范围与说明。内置分类只读；自定义分类保存到本地工作目录 `.ainovel/categories.json`。删除分类不会自动改写已有能力或技能包记录，避免静默改变历史数据。

完整协议见 CAPABILITY_PROTOCOL.md，接口见 API.md。

故障排查

现象原因处理
Pages 显示“当前未连接后端”公开域名正常；只是默认 Pages 部署没有同时运行 Go API浏览器本地功能可继续使用；需要服务端能力时再启动或部署兼容后端
模型测试 401Key 无效或未加载检查环境变量/后台配置并重启服务
模型测试 404Base URL 或 Model 不存在确认 URL 含正确 API 根路径与模型 ID
浏览器报 CORS正在直连第三方 API改为访问同源 Go 后端，不要在 Pages 放 Key
项目消失浏览器站点数据被清除导入最近的项目备份；以后定期导出
旧候选不能替换生成期间正文已变化这是坐标保护；复制候选并手动确认位置

仍未解决？

运行 make check 获取与 CI 一致的检查结果，并在 GitHub Issues 提交复现步骤、终端错误、浏览器控制台信息和运行环境。请先删除日志中的 API Key 与作品隐私内容。
