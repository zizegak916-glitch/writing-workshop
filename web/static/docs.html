使用文档

AI写作工坊是开源可自定义项目：GitHub Pages 发布静态前端，ainovel-cli 的 Go 服务提供可选后端 API。

打开写作工坊
当前 Pages 地址没有常驻后端。你可以浏览界面和文档；真实 AI 调用、配置保存、项目同步和规则管理，需要运行本地后端或部署自己的 API 服务。

快速开始

go build -o ainovel-cli ./cmd/ainovel-cli
./ainovel-cli serve --port 8080

本地服务启动后访问 http://127.0.0.1:8080/app.html 或 http://127.0.0.1:8080/admin.html。此时前端会通过同源 /api/ 端点连接 Go 后端。

核心入口

写作工坊

app.html，用于项目、章节、角色、AI 辅助写作。

管理后台

admin.html，用于配置 provider、model、API key 和项目规则。

API

后端端点以 /api/ 开头，主要包括 /api/ai、/api/config、/api/rules。

开源自定义

模型供应商

在后台或配置文件中修改 provider、model、base URL，也可以通过环境变量注入 API key。

项目规则

通过 /api/rules 管理写作偏好、风格要求和项目约束，规则会落到本地项目目录。

前端页面

静态文件位于 web/static/，fork 后可以直接替换文案、样式和入口页面。

API Key 配置

推荐在管理后台保存 provider 配置，或使用环境变量，例如 AINOVEL_OPENROUTER_API_KEY、OPENROUTER_API_KEY、OPENAI_API_KEY。
