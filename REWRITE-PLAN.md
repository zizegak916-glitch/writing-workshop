# AI 写作工坊 — 大改造计划

> **目标：** 将现有的单体 HTML 应用改造为现代化、模块化、功能完善的写作工具
>
> **当前状态：** 单文件 3475行 / 268KB，IndexedDB v3，12个API服务商，30+ AI模式，i18n，移动端适配
>
> **部署方式：** GitHub Pages（纯前端静态站）

---

## 阶段一：代码架构重构 ⚙️
*把巨型单文件拆成可维护的模块*

### 1.1 拆分文件结构
```
writing-pages/
├── index.html          (入口，只保留骨架)
├── css/
│   ├── variables.css   (CSS变量/主题)
│   ├── layout.css      (布局/响应式)
│   ├── components.css  (组件样式)
│   └── animations.css  (动画/过渡)
├── js/
│   ├── app.js          (主入口/初始化)
│   ├── db.js           (IndexedDB操作)
│   ├── ai.js           (AI调用/API管理)
│   ├── editor.js       (编辑器核心)
│   ├── project.js      (项目/章节/大纲管理)
│   ├── characters.js   (人物系统)
│   ├── import.js       (导入系统)
│   ├── i18n.js         (多语言)
│   └── ui.js           (UI交互/弹窗/Toast)
├── icons/              (SVG图标文件)
└── build.sh            (合并脚本，输出单文件给GH Pages)
```

### 1.2 构建脚本
- `build.sh` 把所有 CSS/JS 合并回单个 `index.html`
- 保持 GitHub Pages 兼容（无构建工具依赖）
- 开发时可直接用多文件调试

## 阶段二：UI/UX 全面升级 🎨

### 2.1 设计系统
- **配色方案**：保留 Linear 风格暗色主题，优化对比度
- **字体**：保留 Inter + Noto Serif SC，优化行高/字间距
- **间距系统**：统一 4px 基准网格
- **圆角/阴影**：统一变量，减少视觉噪音

### 2.2 图标系统
- 替换当前 emoji + 内联 SVG 为 **Lucide Icons**（CDN引入）
- 所有导航、工具栏、按钮使用统一图标
- 保留 AI 模式的自定义 emoji 图标（辨识度高）

### 2.3 布局优化
- **桌面端**：三栏布局（侧栏 | 编辑器 | AI面板），可折叠
- **移动端**：底部 Tab 导航 + 全屏编辑器，滑动切换面板
- **新增**：编辑器全屏模式（隐藏所有面板）
- **新增**：AI面板可独立全屏（移动端友好）

### 2.4 交互动效
- 页面切换淡入淡出
- 按钮 hover/active 微动效
- AI 生成时的流式文字动画
- Toast 通知滑入/滑出

## 阶段三：AI 功能增强 🤖

### 3.1 流式输出（Streaming）
- 支持 SSE（Server-Sent Events）流式响应
- 实时显示 AI 生成内容，不再等待完整返回
- 打字机效果 + 光标闪烁

### 3.2 AI 味审查升级
- **可视化雷达图**：6维度评分用 Canvas/SVG 绘制
- **逐句标注**：高亮 AI 特征明显的句子
- **修改建议**：每个问题给出具体改写建议
- **前后对比**：一键查看修改前后差异

### 3.3 降 AI 率增强
- **多轮降 AI**：支持反复降 AI 直到满意
- **保留度滑块**：控制重写幅度（轻度/中度/重度）
- **选择性降 AI**：只对选中段落生效
- **历史对比**：保存每次降 AI 结果，可回退

### 3.4 真的接 API
- **流式请求**：fetch + ReadableStream 实现
- **错误重试**：自动重试 + 指数退避
- **超时处理**：可配置超时时间
- **多 Key 轮转**：支持配置多个 API Key 自动切换
- **Token 预估**：发送前预估 token 数，超出警告
- **请求队列**：并发控制，避免触发限流

### 3.5 新增 AI 功能
- **AI 续写建议**：在光标位置提供 3 个续写方向
- **风格迁移**：学习用户写作风格，保持一致性
- **角色对话生成**：根据人物档案生成符合性格的对话
- **世界观一致性检查**：检查内容与设定是否矛盾

## 阶段四：导入/导出系统增强 📥

### 4.1 导入增强
- **DOCX 支持**：通过 mammoth.js 解析 Word 文档
- **Markdown 支持**：解析 MD 文件自动分章节
- **批量导入**：支持选择多个文件同时导入
- **URL 导入**：粘贴网页链接自动提取正文
- **剪贴板导入**：支持直接粘贴富文本
- **智能分章**：AI 自动识别章节边界

### 4.2 导出增强
- **Markdown 导出**：保留格式的 MD 文件
- **DOCX 导出**：通过 docx.js 生成 Word 文档
- **PDF 导出**：浏览器打印优化
- **批量导出**：整个项目一键导出
- **导出模板**：可自定义导出格式

### 4.3 数据管理
- **数据迁移**：旧版数据自动升级
- **数据备份**：一键导出所有 IndexedDB 数据
- **数据恢复**：从备份文件恢复
- **数据清理**：清理 AI 历史、过期缓存

## 技术选型

| 用途 | 方案 | 理由 |
|------|------|------|
| 图标 | Lucide Icons (CDN) | 轻量、SVG、Tree-shakeable |
| DOCX 解析 | mammoth.js (CDN) | 成熟、浏览器端运行 |
| DOCX 生成 | docx.js (CDN) | 功能完整 |
| 流式输出 | fetch + ReadableStream | 原生 API，无需依赖 |
| 雷达图 | Canvas 2D | 无依赖，性能好 |
| 构建 | bash 脚本 | 零依赖，GH Pages 兼容 |
