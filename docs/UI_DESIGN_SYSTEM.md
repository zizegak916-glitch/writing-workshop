# Writing Workshop UI 设计系统

> 状态：现行产品视觉规范，更新于 2026-07-22。

Writing Workshop 的界面方向是“彩色编辑部（Ink & Spectrum）”：把长篇创作看作由资料、正文、能力和决策共同组成的编辑工作，而不是把聊天框放大。

![彩色编辑部工作台](images/workbench.jpg)

## 1. 信息分区

| 区域 | 视觉角色 | 产品职责 |
|---|---|---|
| 资料栏 | 深海军蓝、低亮度 | 大纲、章节、人物、笔记；稳定、长期、可扫描 |
| 编辑器 | 暖象牙纸、深色正文 | 唯一正文工作面；安静、长时间可读 |
| AI 区 | 淡紫玻璃、白色卡片 | 任务、上下文、候选、历史；与正文保持明确边界 |
| 顶部栏 | 深色流体玻璃 | 项目切换、统计、保存、导出与后台入口 |

## 2. 色彩语义

- `Ink #15203A`：结构、主操作与正文，不使用纯黑。
- `Paper #FFFAF1`：编辑器和阅读页面。
- `Cobalt #3155D9`：主行动、链接、焦点。
- `Coral #F26B5B`：候选、需要注意的写入动作。
- `Mint #7BD8B2`：已连接、已选择、安全状态。
- `Amber #F2B544`：进度、提醒与导出。
- `Violet #A98DE8`：AI、Skill、推理工作区。

颜色不承担唯一信息通道。选中、警告、连接状态同时使用文字、边框或图标表达。

## 2.1 产品图标

全站使用 `web/static/icons/app-icon.svg`：打开的双色书页代表长篇资料结构，白色笔尖代表作者决策，薄荷与紫色圆点代表可组合能力。图标是仓库原生 SVG，不依赖外部图标服务；favicon、落地页品牌和工作台顶栏引用同一文件。

## 2.2 AI 工作台与能力图标

AI 入口不再使用通用“大脑”图标。新符号的上半部是候选火花，下半部是作者笔尖，表达“模型给出候选，作者决定是否写入”。大脑图标只保留给“记忆”功能，不再混用。

AI 卡片中的 30 个能力使用一对一的仓库原生线性 SVG，覆盖基础、描写、风格、分析、创作和工具六类。不再把 `◇`、`↑`、`♪` 或 Emoji 直接当成图标。六类能力分别使用蓝、珊瑚、紫、薄荷、琥珀和天蓝的图标容器，同时保留分类文字，颜色不是唯一区分方式。

- 图标源：`web/static/icons/ai-mode-icons.svg`
- 能力到图标的映射：`web/static/js/ai-mode-icons.js`
- 卡片分类色与选中态：`web/static/css/chromatic-app.css`

## 3. 交互原则

1. **正文优先。** 中央编辑器始终是桌面最大区域；手机默认打开正文。
2. **候选隔离。** AI 输出使用紫色能力区和独立结果层，不伪装成已写入正文。
3. **动作可逆。** 保存、替换、插入、追加、写入记忆是不同按钮；重要写入保留前置快照。
4. **局部渐进。** 常用动作直接可见，高级参数按任务展开，后台承担 Provider 与能力配置。
5. **状态说人话。** Pages 显示“在线版 · 浏览器本地数据”，服务端可用时显示“后端已连接”，不用“预览”贬低正式域名，也不以模糊红点代替说明。

## 4. 流体玻璃边界

玻璃效果只用于导航、悬浮候选和模态层，内容面保持不透明或高不透明度，避免长文阅读时出现背景干扰。

```css
background: rgba(255, 250, 241, .78);
border: 1px solid rgba(255, 255, 255, .72);
backdrop-filter: blur(24px) saturate(150%);
box-shadow: 0 12px 36px rgba(33, 43, 76, .09);
```

## 5. 响应式规则

- `> 980px`：资料 / 正文 / AI 三栏。
- `768–980px`：缩窄资料和 AI 面板，隐藏次要统计。
- `< 560px`：正文全宽；大纲、章节、人物和 AI 进入底部导航。
- 所有主要点击区域最低 44px；横向工具条允许滚动而不是压缩到不可点击。
- 遵守 `prefers-reduced-motion`，用户关闭动效时移除动画与平滑滚动。

## 6. 代码位置

- 落地页和文档共享：`web/static/css/chromatic-site.css`
- 主工作台后置覆盖：`web/static/css/chromatic-app.css`
- 能力后台：`web/static/css/chromatic-console.css`

工作台使用后置样式层，保留原有 DOM ID、事件与数据操作，避免“重做 UI”破坏导入导出、编辑、项目管理和 AI 工作流。
